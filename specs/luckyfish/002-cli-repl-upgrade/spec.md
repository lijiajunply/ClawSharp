# 功能规范：CLI 交互体验升级 (REPL 2.0)

**功能分支**: `luckyfish/002-cli-repl-upgrade`  
**创建日期**: 2026-03-31  
**状态**: 草案 (Draft)  
**输入**: 用户描述: "CLI 交互体验升级 (REPL 2.0) 当前的 ChatCommand.cs 提供了一个基础的流式输出 REPL，但仍有提升空间： - 增加 /tools 命令：允许用户在对话中查看当前 Agent 可调用的工具列表及其权限状态。 - 多行输入支持：Spectre.Console 的 Ask 默认处理单行。对于编写代码或长 Prompt，需要一个 /paste 模式或更好的多行编辑器集成。 - Session 切换优化：目前的 /resume 只能恢复最后一次，建议支持 /sessions 列表展示并通过编号快速切换。 - 富文本渲染增强：对于 Agent 返回的代码块，可以使用 Spectre.Console 的 SyntaxHighlighter 进行高亮渲染。"

## 用户场景与测试 *(必填)*

### 用户场景 1 - 增强的会话管理 (优先级: P1)

作为一名经常在不同任务间切换的用户，我希望能够查看之前的聊天会话列表，并能快速切换到其中任何一个，以便在不同项目间切换时能保持上下文，而无需重新开始。

**优先级说明**: 对于提高生产力和维护多个上下文至关重要，这是 CLI 的核心用例。

**独立测试**: 可以通过启动多个会话、使用 `/sessions` 列出它们，并使用编号进行切换来完整测试。

**验收场景**:

1. **给定** 存在多个会话，**当** 我输入 `/sessions` 时，**那么** 我应该看到一个格式化的会话列表，包含编号、创建时间以及最后一条消息的预览。
2. **给定** 会话列表，**当** 我输入 `/sessions 2` 时，**那么** CLI 应该加载编号为 2 的会话并显示对话历史。

---

### 用户场景 2 - 代码块语法高亮 (优先级: P1)

作为一名使用 CLI 生成或调试代码的开发人员，我希望 Agent 返回的代码块能够通过语法高亮进行渲染，以便我能更轻松地阅读和理解代码。

**优先级说明**: 对开发人员主要使用的输出格式（代码）的重大体验提升。

**独立测试**: 可以通过让 Agent 生成代码（如 C# 或 Python）并验证终端中是否以彩色渲染来测试。

**验收场景**:

1. **给定** 一个运行中的会话，**当** Agent 返回包含 Markdown 代码块的响应时，**那么** 代码应根据其语言标签使用相应的语法高亮进行渲染。

---

### 用户场景 3 - 工具详情查看 (优先级: P2)

作为一名与专业 Agent 交互的用户，我希望了解当前可用的工具及其权限级别，以便了解 Agent 的能力范围和安全边界。

**优先级说明**: 提高 AI 操作的透明度和信任度。

**独立测试**: 在会话期间调用 `/tools`，并验证输出列表是否与 Agent 的配置一致。

**验收场景**:

1. **给定** 一个配置了特定工具的 Agent 会话，**当** 我输入 `/tools` 时，**那么** 我应该看到工具名称、描述以及当前的权限状态（如：始终允许、询问、拒绝）。

---

### 用户场景 4 - 多行输入支持 (优先级: P2)

作为一名提供复杂提示词或代码片段的用户，我希望在发送给 Agent 之前能够输入多行文本，这样我就不必将所有内容压缩到一行中。

**优先级说明**: 对于代码重构或长系统指令等高级用法是必需的。

**独立测试**: 通过进入“粘贴模式”或“编辑模式”，输入多行文本，并将其作为单个提示词提交来测试。

**验收场景**:

1. **给定** 聊天提示符，**当** 我输入 `/paste` 时，**那么** CLI 应进入多行模式，在该模式下 Enter 键产生新行而不是发送。
2. **给定** 在粘贴模式下，**当** 我输入特定的终止序列（如输入单行 `.`）时，**那么** 整个文本应发送给 Agent。
3. **给定** 聊天提示符，**当** 我输入 `/edit` 时，**那么** CLI 应打开系统默认编辑器，保存并关闭后内容自动发送。

## 功能需求 *(必填)*

### 功能性需求

- **FR-001**: 系统必须提供 `/tools` 命令，列出当前 Agent 注册的所有工具。
- **FR-002**: `/tools` 输出必须包含工具名称、简短描述及其权限状态。
- **FR-003**: 系统必须支持 `/sessions` 命令，显示存储在 SQLite 数据库中的历史会话列表。
- **FR-004**: 系统必须允许通过编号或 ID（如 `/sessions <id>`）切换到特定会话。
- **FR-005**: REPL 必须使用语法高亮组件（如 Spectre.Console 的 `SyntaxHighlighter`）渲染 Agent 响应中的 Markdown 代码块。
- **FR-006**: 系统必须实现双重多行输入机制：
    - 提供 `/paste` 命令用于终端内多行输入。用户进入该模式后，系统需捕获所有输入，直到输入**仅包含一个句点 (`.`) 的单行**为止。
    - 提供 `/edit` 命令用于调用系统默认编辑器（通过 `EDITOR` 环境变量，Windows 默认为 `notepad`）进行复杂编辑。保存并退出编辑器后，内容自动作为消息发送。
- **FR-007**: 系统必须在切换单行和多行输入模式时提供清晰的视觉反馈。在 `/paste` 模式下，左侧提示符应从 `Agent >` 变为 `Paste >`。

### 核心实体 *(涉及数据时包含)*

- **会话 (Session)**: 代表一个唯一的聊天记录，包括其历史、元数据（开始时间、最后更新时间）和关联的 Agent。
- **工具定义 (Tool Definition)**: 有关运行时可用工具的元数据，包括其来源（本地/MCP）和用户定义的权限级别。

## 成功指标 *(必填)*

### 可衡量的结果

- **SC-001**: 用户列出并切换会话的时间应在 5 秒以内。
- **SC-002**: 100% 带有标准语言标签（C#, JS, Python, Go, Rust 等）的代码块能被语法高亮渲染。
- **SC-003**: `/tools` 命令能立即准确反映工具可用性或权限的变化。
- **SC-004**: 用户能够使用多行支持功能成功输入并发送超过 10 行的提示词。

## 假设条件

- **现有持久化**: 现有的基于 SQLite 的会话存储足够健壮，只需少量修改即可支持 `/sessions` 列表功能。
- **终端能力**: 用户的终端支持 Spectre.Console 所需的 ANSI 转义序列（颜色和格式）。
- **工具元数据**: 工具描述和权限状态已经可以通过 `IClawKernel` 或 `IClawRuntime` 访问。

## 边界情况

- **会话列表溢出**: 当会话超过 20 个时，仅显示最近的 20 个，并提示用户使用更具体的查询或 ID。
- **缺失语言标签**: 对于没有语言标签的代码块，使用带有浅灰色背景的纯文本进行渲染。
- **超大粘贴输入**: 当输入内容超过 10,000 个字符时，系统应在发送前弹出警告确认，告知用户可能导致响应延迟。
