# 功能规格说明书：DuckDB 分析层落地与 `claw stats` 命令

**功能分支**: `luckyfish/002-claw-stats-duckdb`  
**创建日期**: 2026-03-31  
**状态**: 草案  
**输入**: 用户描述: "3. 数据持久化与分析的闭环 - DuckDB 分析层落地：代码中已有 IDuckDbAnalyticsProjector 的占位和基础实现，但目前 CLI 还没有命令可以展示这些数据。 - 改进建议：增加 claw stats 命令，利用 DuckDB 查询 Session 的 Token 消耗趋势、工具调用频率以及各 Agent 的响应时长分布。"

## 用户场景与测试 *(强制)*

### 用户场景 1 - 查看资源消耗概览 (优先级: P1)

用户希望通过一个命令快速了解最近使用 ClawSharp 产生的资源消耗情况，包括总 Token 数和估算的成本。

**为什么是此优先级**: 这是分析功能的核心价值，帮助用户控制成本和了解使用强度。

**独立测试**: 运行 `claw stats`，应能从 DuckDB 中提取并显示累计 Token 消耗数据。

**验收场景**:

1. **给定** 存在历史 Session 数据且已投影至 DuckDB，**当** 用户运行 `claw stats` 时，**那么** 系统应显示总消耗 Token（Prompt 和 Completion）。
2. **给定** 系统运行正常，**当** 用户执行命令时，**那么** 响应时间应在 2 秒内。

---

### 用户场景 2 - 分析工具调用偏好 (优先级: P2)

用户希望了解哪些工具被频繁调用，哪些工具可能存在问题或不常使用。

**为什么是此优先级**: 帮助开发者优化工具定义和 Agent 提示词。

**独立测试**: 运行 `claw stats --tools`，应列出所有已调用工具的频率排名。

**验收场景**:

1. **给定** 有工具调用记录，**当** 执行统计命令时，**那么** 应按调用次数降序排列显示工具名称及其成功/失败次数。

---

### 用户场景 3 - 监控 Agent 性能趋势 (优先级: P3)

用户（开发者）希望了解不同 Agent 的平均响应时长分布，以识别性能瓶颈。

**为什么是此优先级**: 用于系统调优和性能分析。

**独立测试**: 运行 `claw stats --agents`，应显示各 Agent 的平均响应时间和响应时间百分位数。

**验收场景**:

1. **给定** 多个 Agent 的交互记录，**当** 查询性能指标时，**那么** 应显示每个 Agent 的平均延迟（秒）和请求总数。

---

## 边界情况

- **无数据状态**: 当 DuckDB 数据库为空或尚未进行任何交互时，命令应友好提示“暂无统计数据”。
- **时间范围筛选**: 用户请求超出实际数据覆盖的时间范围。
- **DuckDB 连接异常**: 如果 DuckDB 文件被占用或无法访问，应提供清晰的错误信息而非崩溃。

## 功能需求 *(强制)*

### 功能需求清单

- **FR-001**: 系统必须提供 `claw stats` 基础命令，展示汇总统计数据。
- **FR-002**: 系统必须支持 `--period` 参数（如 24h, 7d, 30d）进行时间范围筛选。
- **FR-003**: 系统必须从 DuckDB 分析层查询 Token 消耗趋势数据。
- **FR-004**: 系统必须统计并展示工具调用成功率与频率。
- **FR-005**: 系统必须计算并展示各 Agent 的平均响应时长分布。
- **FR-006**: 统计输出必须支持表格化展示，并利用颜色区分正常与异常状态。
- **FR-007**: 系统必须确保统计查询不会影响正在进行的 AI 会话性能。

### 关键实体 *(涉及数据时包含)*

- **TokenUsageMetric**: 代表 Token 消耗的时序数据或汇总，包含输入/输出 Token 数和时间戳。
- **ToolMetric**: 代表特定工具的运行指标，包含工具名、调用次数、成功数、失败数。
- **AgentPerformance**: 代表特定 Agent 的性能指标，包含 Agent ID、平均延迟、最大延迟。

## 成功标准 *(强制)*

### 可衡量结果

- **SC-001**: 用户在执行 `claw stats` 后 2 秒内看到统计结果。
- **SC-002**: 统计数据的准确率必须达到 100%（与底层 Session 日志一致）。
- **SC-003**: 统计输出在 80 列宽度的终端中能清晰显示，不出现错行（通过自动列宽缩减或长文本截断实现）。
- **SC-004**: 提供至少 3 个维度的统计信息：成本(Tokens)、效率(Latency)、活跃度(Tool usage)。

## 假设

- 用户机器已安装必要的运行时以支持 DuckDB（通常通过 NuGet 包内嵌）。
- `IDuckDbAnalyticsProjector` 已能正确将底层消息/事件实时同步到 DuckDB。
- 目标用户具备基本的 CLI 操作常识。
- 目前仅支持本地 DuckDB 分析，暂不考虑远程分析后端。
