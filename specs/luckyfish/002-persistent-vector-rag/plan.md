# Implementation Plan: Persistent Vector Store and Automated RAG

**Branch**: `luckyfish/002-persistent-vector-rag` | **Date**: 2026-03-31 | **Spec**: [/Users/luckyfish/Documents/Project/RiderProjects/ClawSharp/specs/luckyfish/002-persistent-vector-rag/spec.md]
**Input**: Feature specification from `/specs/luckyfish/002-persistent-vector-rag/spec.md`

## Summary

实现基于 SQLite-VSS 的持久化向量存储，替换目前的 `InMemoryVectorStore`。引入真实的 Embedding 提供者（OpenAI 和本地 ONNX/FastEmbed），并在 `ClawRuntime` 中实现自动化的 RAG 注入逻辑。

## Technical Context

**Language/Version**: C# 14 / .NET 10  
**Primary Dependencies**: `Microsoft.Extensions.DependencyInjection`, `Sqlite-VSS` (extension), `Microsoft.ML.OnnxRuntime` (for local embedding) or `FastEmbed.Net`  
**Storage**: SQLite with VSS extension for vector data; EF Core for metadata  
**Testing**: xUnit, `ClawSharp.Lib.Tests` (Integration tests)  
**Target Platform**: Windows, macOS, Linux (Local-first)
**Project Type**: Library (`ClawSharp.Lib`) and CLI (`ClawSharp.CLI`)  
**Performance Goals**: RAG injection latency < 500ms (Local)  
**Constraints**: < 200ms p95 for vector search, offline-capable for local embedding  
**Scale/Scope**: Support thousands of knowledge chunks per Agent/Session

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- [x] **Library-First**: Logic will be implemented in `ClawSharp.Lib`.
- [x] **Local-First**: Support for local Embedding (ONNX) and local vector store (SQLite).
- [x] **Async-First**: All IO and model calls will use `Task`/`ValueTask`.
- [x] **Extensibility**: `IEmbeddingProvider` and `IVectorStore` abstractions will be used.
- [x] **Reliable Persistence**: Vector data will be stored on disk via SQLite.
