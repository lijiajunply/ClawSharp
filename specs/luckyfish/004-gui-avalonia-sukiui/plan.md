# Implementation Plan: Add Avalonia+SukiUI GUI version

**Branch**: `luckyfish/004-gui-avalonia-sukiui` | **Date**: 2026-04-02 | **Spec**: [/specs/luckyfish/004-gui-avalonia-sukiui/spec.md](/specs/luckyfish/004-gui-avalonia-sukiui/spec.md)
**Input**: Feature specification from `/specs/luckyfish/004-gui-avalonia-sukiui/spec.md`

## Summary

Implement a cross-platform desktop GUI for ClawSharp using Avalonia and SukiUI. The application will serve as a thin client for `ClawSharp.Lib`, providing a modern chat interface, theme management, and agent selection. Technical approach involves MVVM pattern with `ReactiveUI` (standard for Avalonia), integrating `IClawKernel` for agent interactions, and leveraging SukiUI for a polished look and feel.

## Technical Context

**Language/Version**: C# 14 / .NET 10  
**Primary Dependencies**: Avalonia (11.x), SukiUI, ClawSharp.Lib, ReactiveUI  
**Storage**: SQLite (via ClawSharp.Lib)  
**Testing**: xUnit, Avalonia.Headless (for UI unit tests)  
**Target Platform**: Windows, macOS, Linux (Desktop)
**Project Type**: Desktop Application  
**Performance Goals**: Main window load < 3s, UI rendering < 100ms, Theme switch < 200ms  
**Constraints**: Must follow "Library-First" (no core logic in Desktop project), Async-first for all kernel interactions.  
**Scale/Scope**: Initial MVP with chat, agent list, and theme toggle.

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- **Library-First**: PASS. `ClawSharp.Desktop` will only contain UI/View logic, delegating all AI and state management to `ClawSharp.Lib`.
- **Markdown-Driven**: PASS. The UI will display and allow selection of agents defined via Markdown in the workspace.
- **Async-First**: PASS. All kernel calls and I/O will be performed asynchronously to keep the UI responsive.
- **Provider & Tool Extensibility**: PASS. GUI will leverage the existing extensible provider/tool system in the library.
- **Reliable Persistence**: PASS. Chat history and session data will be persisted using the library's SQLite backend.
