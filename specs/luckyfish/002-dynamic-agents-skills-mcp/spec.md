# 功能说明书：动态 Agent、技能录入及 MCP 支持

**功能分支**: `luckyfish/002-dynamic-agents-skills-mcp`  
**创建日期**: 2026-03-30  
**状态**: 草案  
**输入**: 用户描述："动态录入 agent (访问~/.agent目录);动态录入 skills (访问 ~/.skills 目录);以及 MCP 支持"

## 用户场景与测试 *(必填)*

### 用户场景 1 - 本地 Agent 自动发现 (优先级: P1)

作为开发者，我希望将 Agent 定义文件放入用户目录下的 agent 文件夹中，以便在 AI 会话中立即使用它，而无需修改项目源代码。

**为什么是此优先级**: 支持 AI 行为的快速原型设计和个性化，这是“本地优先”AI 内核的核心价值。

**独立测试**: 在 `~/.agent/` 中放置一个有效的 `custom-agent.md` 文件，并验证 `ClawSharp.CLI` 能在可用 Agent 列表中显示它。

**验收场景**:

1. **给定** `~/.agent/` 中存在有效的 Agent Markdown 文件，**当**内核启动时，**那么**该 Agent 应注册到 `AgentRegistry` 中。
2. **给定**一个无效的 Markdown 文件（缺少 YAML 前置配置），**当**内核扫描目录时，**那么**系统应记录警告并继续加载其他有效的 Agent。

---

### 用户场景 2 - 本地技能扩展 (优先级: P2)

作为高级用户，我希望在中心位置（`~/.skills/`）维护一个个人 AI 技能库，这些技能可以被使用 ClawSharp 内核的不同项目共享。

**为什么是此优先级**: 促进复杂工具定义的重用，并简化自定义功能的管理。

**独立测试**: 在 `~/.skills/` 中放置一个 `web-search.md` 技能，并验证 Agent 在聊天会话中可以调用其中定义的工具。

**验收场景**:

1. **给定** `~/.skills/` 中有多个技能文件，**当**内核初始化时，**那么**所有有效的技能都应合并到全局 `SkillRegistry` 中。
2. **给定** `~/.skills/` 中的技能与内置技能同名，**当**加载时，**那么**系统应自动为本地技能添加命名空间前缀（例如 `user.`）以避免冲突，并允许两者共存。

---

### 用户场景 3 - MCP 工具集成 (优先级: P1)

作为 AI 工程师，我希望将 ClawSharp 连接到外部模型上下文协议（MCP）服务器，以便我的 Agent 可以访问托管在内核之外的专业工具（如数据库浏览器或云 API）。

**为什么是此优先级**: MCP 是工具互操作性的行业标准；支持该协议对生态系统兼容性至关重要。

**独立测试**: 配置一个“文件系统”MCP 服务器，并验证 AI 能使用该服务器提供的工具列出文件并读取内容。

**验收场景**:

1. **给定**已配置的 MCP 服务器（stdio 传输），**当**会话启动时，**那么**内核建立连接并获取工具列表。
2. **给定**一个离线的 MCP 服务器，**当** AI 尝试调用其工具时，**那么**系统应向 AI 返回友好的错误提示。

---

### 边界情况

- **路径解析**: 如果 `~/.agent` 或 `~/.skills` 目录不存在，系统如何处理？（假设：自动创建或静默忽略）。
- **安全性**: 如何防止恶意的 Agent/技能文件执行未经授权的 Shell 命令？（假设：依赖现有的工具安全策略）。
- **冲突管理**: 两个 MCP 服务器提供同名的工具。

## 需求 *(必填)*

### 功能需求

- **FR-001**: 系统 MUST 在启动时扫描 `~/.agent/` 目录下的 `.md` 文件。
- **FR-002**: 系统 MUST 在启动时扫描 `~/.skills/` 目录下的 `.md` 文件。
- **FR-003**: 系统 MUST 支持 MCP `stdio` 传输协议以连接本地 MCP 服务器。
- **FR-004**: 系统 MUST 允许用户通过本地配置文件（例如 `~/.clawsharp/mcp.json`）配置 MCP 服务器。
- **FR-005**: 系统 MUST 提供一种机制，无需重启应用程序即可刷新动态注册表。
- **FR-006**: 系统 MUST 在初期版本中优先支持 MCP `stdio` 传输；远程服务器支持（SSE/HTTP）应标记为未来增强功能。

### 关键实体 *(如果涉及数据)*

- **动态 Agent 定义**: 位于用户主目录中的带有 YAML 前置配置的 Markdown 文件。
- **MCP 服务器配置**: 启动和连接 MCP 服务器所需的元数据（命令、参数、环境变量）。
- **MCP 工具代理**: 内核级包装器，负责在 ClawSharp 工具调用和 MCP 协议消息之间进行转换。

## 成功标准 *(必填)*

### 可衡量结果

- **SC-001**: 用户保存文件到 `~/.agent/` 后，5 秒内（不含文件系统延迟）即可在 UI 中看到该 Agent。
- **SC-002**: 系统能同时成功连接并执行至少 3 个并发 MCP 服务器的工具。
- **SC-003**: 目标目录中 100% 的有效 Markdown Agent 均能被加载，无需手动配置。

## 假设

- **主目录**: 符号 `~` 能正确解析为特定操作系统的用户配置文件目录（如 `/Users/name`）。
- **Markdown 模式**: 动态 Agent 和技能使用与内置组件相同的 YAML 前置配置架构。
- **MCP 规范**: 实现目标为 MCP 1.0 规范。
- **配置**: MCP 服务器设置以 JSON 格式存储在用户主目录的隐藏文件夹中。
